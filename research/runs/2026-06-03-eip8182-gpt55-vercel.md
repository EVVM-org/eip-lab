## Judge verdict: best run to date — one character from compiling

### Strengths (the bar moved)
- **Single-shot, complete.** 24,753 completion tokens covered all 8 files
  including a full `justification.md`, with no continue turns — so NO
  continue-boundary welds (the dominant defect in the deepseek run).
- **Correct incremental Merkle tree.** `_insertNote` implements the
  standard filled-subtrees/frontier append (Tornado-style) correctly, with
  no narration and no fabricated zero-ladder — the empty ladders are
  derived in the constructor over the keccak mock. This is exactly the
  code deepseek spiralled on and then fabricated.
- **Parameter-free Poseidon mock**, clean `hash2..5`/`domain` helpers — the
  anti-fabrication rule holds on a third distinct model.
- **All interfaces file-scope and self-contained**; no in-contract
  interface, no bogus `override` (both were deepseek compile-blockers).
- Faithful surface: `deposit` / `transact` / `setAuthPolicy` + read
  methods; 19-field public-input vector; range checks; pool-proof and
  auth-proof dispatch via `staticcall` with `ok && ret.length==32 &&
  decode(bool)` checks; ERC-20 balance-delta on deposit and return-shape
  handling (empty-return-with-code OR 32-byte true); reentrancy guard;
  `receive()` reverts to force `deposit`; block-based auth-root snapshot.
- Honest `Limitations:` notes on every mock; complete justification.

### The single compile-blocker
- **`MockAuthVerifier8182.sol` constructor is missing its opening brace:**
  ```solidity
  constructor(address admin_)   require(admin_ != address(0), "EIP8182: zero admin");
      admin = admin_;
  }
  ```
  A single dropped `{` between `)` and `require` — the same weld class seen
  on other models, but here reduced from whole missing functions to ONE
  character in ONE file. With the brace restored the file is well-formed.

### Minor / non-blocking
- `using MockPoseidon8182 for uint256[];` is declared but the code calls
  the library functions directly — harmless dead `using`.
- `_validatePublicInputRanges` re-checks `< FIELD` for all 19 inputs
  including addresses already bounded to 2^160 — redundant, not wrong.
- Logic fidelity caveats remain inherent to the mock approach (the pool
  proof is an admin verdict, so membership/value-conservation are trusted),
  which the contract states explicitly.

### One-line judgment
The strongest artifact in the dataset: single-shot, no fabrication, no
spiral, correct frontier Merkle, self-contained — blocked from compiling
only by a single dropped `{` in one mock's constructor.

| **gpt-5.5** | **OpenAI** | **$1.58** | **no** | **one dropped `{`** | **one character** |

- **Quality scaled with price here, and with single-shot completion.**
  gpt-5.5 cost ~3.5× deepseek and ~21× qwen3-coder, and produced the
  closest-to-compilable, most faithful output — largely because it
  finished in ONE turn (24.7k completion) and so never hit the
  continue-weld failure mode that gutted the deepseek run.
- **Continuation is still the dominant cross-model risk, and avoiding it
  is the biggest quality lever.** The model that didn't need to continue
  produced the cleanest result. This reinforces the deferred harness fix
  (file-aware continuation) AND argues for raising single-shot budget /
  pushing concision so large EIPs finish in one turn.
- **The weld class is now down to one character.** Even gpt-5.5 dropped a
  single `{` — consistent with a residual mid-stream/`<think>`-adjacent
  splice rather than a model logic error. A trivial post-generation
  "balance braces / lint" pass, or the file-aware continuation fix, would
  likely have made this run compile outright.
- **OpenAI gives exact usage.** Unlike Venice streaming (which omits the
  usage chunk and forces a ~4-chars/token estimate), OpenAI returns usage,
  so this row's tokens/cost are measured, not estimated — useful as the
  calibration point for the estimated Venice rows.
